Accessibility statement

This accessibility statement applies to mymca-prod.powerappsportals.com/book_and_manage_an_oral_exam/. This website is run by The Maritime and Coastguard Agency (MCA) and is designed to be used by seafarers.

How you should be able to use this website

We want as many people as possible to be able to use this website. You should be able to:

  • change colours, contrast levels and fonts using browser or device settings
  • zoom in up to 400% without the text spilling off the screen
  • navigate most of the website using a keyboard or speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

We’ve also made the website text as simple as possible to understand.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

Parts of this website are not fully accessible. For example:

  • some form labels are missing or incorrect
  • some aria-controls used with a date picker toggle may reference elements that cannot be verified
  • some input fields do not include valid autocomplete attributes where expected
  • some abbreviations are not pronounced as intended

The Maritime and Coastguard Agency (MCA) service

Each stage of the service; Online Oral Exam (OOE) and Certificate of Competency (CoC) has its own accessibility statement that details how accessible the service is, how to report problems, and how to request information in an alternative format. You can access these pages from the footer inside each service.

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’).

If you raise a complaint and you’re not happy with our response, contact the Equality Advisory and Support Service (EASS)

Technical information about this website’s accessibility

The Maritime and Coastguard Agency are committed to making this website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines (WCAG) 2. 2 AA standard, due to the following non-compliances and exemptions.

Non-accessible content

The following content is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

  • Screen readers may announce the field incorrectly or not at all, and Voice Control users may be unable to select fields using spoken labels. Affected WCAG criteria: SC 4.1.2 and SC 2.5.3, WCAG 2.0A & 2.1A. Some form labels are missing or incorrect. Some form fields do not have properly associated labels, or the visible label does not match the programmatic (accessible) name.
  • Screen reader users may receive misleading information about component states or relationships Invalid ARIA usage. Affected WCAG criteria: SC 4.1.2, WCAG 2.0A. Some ARIA attributes (for example, aria-controls used with a date picker toggle) may reference elements that cannot be verified.
  • Users may need to manually re-enter information, increasing effort and risk of errors. Affected WCAG criteria: SC 1.3.5, WCAG 2.1AA. There are missing or invalid autocomplete attributes, and certain input fields do not include valid autocomplete attributes where expected. While this does not prevent use of the service, it impacts autofill or cognitive support to some level.
  • Users may hear content pronounced in unintended ways. Affected WCAG criteria: WCAG 2.1AA WCAG 2.2 AA. When reading content, Voice Control fails to pronounce abbreviations as intended; hence users will hear it as sound rather than an abbreviation.

Content that’s not within the scope of the accessibility regulations

Non-HTML documents published before September 2018 do not need to be accessible unless users need them to use a service.

What we’re doing to improve accessibility

We are addressing accessibility limitations and will update the accessibility statement when issues are fixed or with information that details when we expect them to be fixed.

How we tested this service

Accessibility testing was conducted using Axe, Browser Stack, NVDA and Mac VoiceOver tools, which tested a subset of Web Content Accessibility Guidelines (WCAG 2.2) Level A and AA success criteria. Results from automated testing were supplemented with manual evaluation in accordance with the Web Content Accessibility Guidelines (WCAG‑EM) V2. 2, Level A and AA methodology to evaluate overall conformance.
